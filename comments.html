COMMENTS

Comments are lines of text within code that the interpreter ignores during execution. They are essential for improving code readability, helping developers understand the purpose, logic, and flow of the code. Properly commented code is easier to maintain, debug, and collaborate on, especially in larger projects.

There are two types of Comments.

  • Single line Comments
  • Multi Line Comments also known as Doc String

Single Line Comments

Single-line comments start with a hash symbol (#) and continue to the end of the line. The Python interpreter ignores everything after the # symbol on that line. Single-line comments are often used to provide brief explanations of code lines or to disable code temporarily. 

# Single Line Comment

Multi-Line Comments

Python does not have a dedicated syntax for multi-line comments like some other programming languages. However, multi-line comments can be achieved Using triple quotes ( ' ' ' or " " " ) as block comments. Triple quotes ( ' ' ' or " " " ) are primarily used for docstrings but can also be used to comment out multiple lines if they are not assigned to any variable. Although technically a string, Python ignores it if it's not assigned, effectively making it a comment.

''' 
this is a 
multi line 
comment
'''

Note: Triple-quoted comments are not officially comments; they are strings. But since they are ignored when not assigned, they behave like comments. However, many developers prefer using # for multi-line comments for clarity.
