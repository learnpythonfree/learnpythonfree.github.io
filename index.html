VARIABLES ⇧

A variable is like a container in programming that holds a value. You can think of it as a box where you can store information that you want to use later in your code.

Why Use Variables?


Variables are important because they allow you to: Store data that can change while your program is running (like a score in a game). Make your code easier to read and manage. Avoid repeating the same value multiple times in your code.
Naming Variables: You can name a variable almost anything you want, but there are some rules: It must start with a letter (a-z, A-Z) or an underscore (_). It can contain letters, numbers (0-9), and underscores, but no spaces. It cannot start with a number. Variable names are case-sensitive (e.g., myVariable and myvariable are different)

You assign a value to a variable using the = sign. For example:.

COMMENTS ⇧

Comments are notes in your code that are ignored by the Python interpreter when the code runs. They are used to explain what your code does, making it easier for you and others to understand.

Why Use Comments?


Clarification: They help explain complex parts of the code.
Documentation: They provide context for your code, which is useful when you or someone else comes back to it later.
Debugging: You can comment out lines of code to prevent them from running, which can be helpful for testing.

LISTS ⇧

A list is a way to store a bunch of values together in one place so you can access, change, and work with them easily. Imagine a list like a container that holds multiple items in a specific order. You can put anything in a list: numbers, words, or even other lists!

Basic List Concept


In Python, a list is written with square brackets [ ], and items are separated by commas. Here's a simple example of a list: 

            
                fruits =
                ["apple", "banana", "mango"]

Here, fruits is a list with three items: "apple", "banana", and "mango".

Key Points about Lists

Ordered: The order of items in a list matters. The first item is at position 0, the second is at position 1, and so on.
Changeable: You can add, remove, or change items in a list after creating it.
Allows Duplicates: Lists can have the same item more than once.

You can get an item by its position (or "index"): 

                
                    print(fruits[0]) # Output -> apple 

                    print(fruits[1]) # Output -> banana

You can add an item to the end of a list with .append(): 

                
                    fruit.append("orange") 
# Now fruits is 
 ["apple", "banana", "cherry", "orange"]

You can remove an item by its value with .remove(): 

                
                    fruit.remove("banana") 
 # Now fruits is 
 ["apple", "cherry", "orange"]

TUPLES ⇧

A tuple is a collection of items, similar to a list, but ordered and immutable (meaning it can't be changed after creation).

Basic Tuples Concept


You create a tuple by placing items inside parentheses () separated by commas, like this: 

            
                my_tuple=(1, 2, 3, "apple", "banana")

Each item in a tuple can be of any data type (integers, strings, lists, other tuples, etc.).

Key Points about Tuples

Ordered: The order of items in a tuple matters. The first item is at position 0, the second is at position 1, and so on.
Immutable(means unchangeble): You cannot add, remove, or change items in a tuple after creating it.
Allows Duplicates: Tuples can have the same item more than once.

You can get an item by its position (or "index") like LISTS: 

                
                    print(fruits[0]) 
 # Output -> apple 

                    print(fruits[1]) 
 # Output -> banana

You cannot add an item to the tuple 

                
                    my_tuple.append("orange") 
# This will raise an error

You cannot remove any item in tuple 

                
                    my_tuple.remove("banana") 
 # This will raise an error

When to Use a Tuple

You want to store a fixed collection of items that shouldn't change. You need data integrity, ensuring that values won't accidentally change. You want a lightweight, faster alternative to lists for read-only data.
