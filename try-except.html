TRY-EXCEPT

Python's try-except blocks are essential for handling errors gracefully. Errors, like invalid user input or missing files, can crash programs. With try-except, you can catch these errors and respond without interrupting your code.

Basic Structure of Try-Except

Here‚Äôs the basic format:

try:
    # Code that might cause an error
except ErrorType:
    # Code to handle the error if it happens

Example:

try:
    num = int(input("Enter a number: "))  # Try to convert input to an integer
    print("Your number is:", num)
except ValueError:
    print("Invalid input! Please enter a number.")

Handling Multiple Exceptions

You can catch different types of errors separately by using multiple except blocks:

try:
    num = int(input("Enter a number: "))
    result = 10 / num
except ValueError:
    print("That's not a valid number!")
except ZeroDivisionError:
    print("You can't divide by zero!")

Using a General Exception

If you‚Äôre unsure about the specific error type, you can use a general Exception to catch all errors. It‚Äôs useful for debugging, but avoid it unless necessary.

try:
    num = int(input("Enter a number: "))
    result = 10 / num
except Exception as e:
    print("An error occurred:", e)

Finally Block

The finally block always executes, whether an error occurs or not, making it useful for cleanup.

try:
    file = open("data.txt", "r")
    data = file.read()
except FileNotFoundError:
    print("File not found.")
finally:
    file.close()
    print("File closed.")

Summary

  • The try block runs code that might raise an error.
  • Each except block catches a specific error type.
  • finally always runs, ensuring resources are cleaned up.

Example: Full Try-Except Block

try:
    num = int(input("Enter a number: "))
    result = 100 / num
    print("Result:", result)
except ValueError:
    print("Please enter a valid integer.")
except ZeroDivisionError:
    print("Cannot divide by zero.")
finally:
    print("End of try-except block.")
